1. Linux 下基于 QEMU 的纯净 Android 虚拟机（KVM と VirGL、小足印）

1.1. 写在前面

最近有在电脑上玩明日方舟的需求，因为是 Linux 电脑、模拟 iOS 是不现实的，所以打算搭一个安卓虚拟机。至于为什么不用 WayDroid，主要是因为它的 Makefile 似乎只支持 SystemD 系统、同时需要内核支持（显得十分肮脏）。于是我还是用老办法，使用 QEMU 模拟之。

本文所写的方法仅依赖 QEMU（需有 KVM 和 VirGL 及 SDL 支持），能够在 X 和 Wayland 环境下正常工作。笔者使用的系统为 Gentoo、故支持 QEMU 内 UEFI 启动的 OVMF 默认安装路径可能跟你的会有不同。

1.2. 安装有 KVM／VirGL 和 SDL 支持的 QEMU（非 Gentoo 用家可跳过）

1.2.1. 本体

在 Gentoo 下，安装​app-emulation/qemu​，并确保其启用了以下两个 USE 旗标： 

+ + sdl                               : Enable the SDL-based console
+ + virgl                             : Enable experimental Virgil 3d (virtual software GPU)

1.2.2. EDK II OVMF

为了支持 UEFI，我们还需要安装​sys-firmware/edk2-ovmf​。它会被安装至路径： 

total 6.2M
-rw-r--r-- 1 root root  16K Aug 28  2022 EnrollDefaultKeys.efi
-rw-r--r-- 1 root root 1.9M Aug 28  2022 OVMF_CODE.fd
-rw-r--r-- 1 root root 1.9M Aug 28  2022 OVMF_CODE.secboot.fd
-rw-r--r-- 1 root root 128K Aug 28  2022 OVMF_VARS.fd
-rw-r--r-- 1 root root 887K Aug 28  2022 Shell.efi
-rw-r--r-- 1 root root 1.5M Aug 28  2022 UefiShell.img

1.3. 取得 Android-x86 系统镜像

于https://sourceforge.net/projects/android-x86/files/下载系统镜像。除该项目之外，你还可以选择其他的项目，如开发更活跃、引入新特性更多的 BlissOS。

1.4. 安装虚拟机

1.4.1. 启动脚本

新建一个文件夹，将取得的系统镜像重命名为​android-x86.iso​后存入该路径。随后在该文件夹中创建名为​LaunchD.sh​的脚本，内容如下： 

#!/usr/bin/sh

VM_PATH="/home/$(whoami)/Android"

args=(
  -enable-kvm
  -bios "/usr/share/edk2-ovmf/OVMF_CODE.fd"
  -m 8G
  -smp $(($(nproc)/2))
  -device intel-hda
  -device hda-duplex
  -device virtio-vga-gl
  -display sdl,gl=on
  -hda "$VM_PATH/android.img"
  -cdrom "$VM_PATH/android-x86.iso"
)

qemu-system-x86_64 "${args[@]}"

其中需要注意的是：

  • bios​指定 OVMF 的路径、可能需要根据你的发行版安装的位置进行调整；
  • m​和​smp​请根据实际情况调整；
  • intel-hda​和​​hda-duplex​为音频系统相关。

保存并写入脚本后，给予它执行权限： 

chmod +x LaunchD.sh

1.4.2. 创建虚拟磁盘

为存放我们的 Android 虚拟机，我们需要创建一个虚拟磁盘，如以下命令以创建一个 60G 的磁盘： 

qemu-img create -f qcow2 android.img 60G

需要注意，​60G​为该磁盘的最大大小，也即该文件的最大大小；文件的实际大小会随其中实际装载的数据的大小而变化。

1.4.3. 安装

运行脚本启动该虚拟机。按照官方指南安装即可。

1.5. 优化／配置

1.5.1. 调整屏幕分辨率／尺寸参数

首先如下修改启动脚本，添加窗口尺寸参数： 

<   -device virtio-vga-gl,xres=2160,yres=1440
---
>   -device virtio-vga-gl

随后启动虚拟机，并于启动加载器选择 DEBUG 模式，进入 MirBSD Korn Shell 环境。随后在其中执行： 

mkdir /boot/
mount /dev/sda1 /boot/
vi /boot/efi/boot/android.cfg

进行以下修改： 

@@ -10,7 +10,7 @@
   set kd=$2
   shift 3
-  linux $kd/kernel root=/dev/ram0 $src $@
+  linux $kd/kernel root=/dev/ram0 $src $@ nomodeset video=2160*1440
   initrd $kd/initrd.img

@@ -49,6 +49,7 @@
   loadfont DejaVuSansMono-18
+  set gfxmode=2160x1440
   terminal_output gfxterm
   set theme=$prefix/theme/theme.txt

最后清理： 

umount /dev/sda1
rm -r /boot/
exit

1.6. 写在最后

如此，我们可以实现一个较高性能的安卓虚拟机，而且是在较为干净的情况下。至于为什么不用 GTK 显示前端，因为它不好！
